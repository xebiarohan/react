1. Server actions 
   - We can create server actions inside the server components
   - These function will get executed only on the server.
   - Function must be async
   - We cannot have server actions in the client components

``` example.js
export default function ShareMealPage() {
  async function shareMeal(formData) {
    "use server";
    const meal = {
        title: formData.get('title'),
        summary: formData.get('summary'),
        instuctions: formData.get('instructions'),
        image: formData.get('image'),
        creator: formData.get('name'),
        creator_email: formData.get('email')
    }

    console.log(meal);
  }

...
} 

```

3. Using server actions in client component
   - We cannot keep the server actions in the client component
   - We have to create a separate file that contains all the server actions
   - Then we can import it in the client component
   - In the above example we don't need to change the value of `action` property, we just have to move the `shareMeal` function.
   - Here we created a separate `actions.js` file and added this server action in it.

```actions.js
"use server";

export async function shareMeal(formData) {
  const meal = {
    title: formData.get("title"),
    summary: formData.get("summary"),
    instuctions: formData.get("instructions"),
    image: formData.get("image"),
    creator: formData.get("name"),
    creator_email: formData.get("email"),
  };

  console.log(meal);
}


```

5. Redirect 
   - Once we completed any action and move to a different page like submitting some form
   - we can use redirect from `next/navigation`

``` example.js
import { redirect } from "next/navigation";
import { saveMeal } from "./meals";

export async function shareMeal(formData) {
  const meal = {
    title: formData.get("title"),
    summary: formData.get("summary"),
    instuctions: formData.get("instructions"),
    image: formData.get("image"),
    creator: formData.get("name"),
    creator_email: formData.get("email"),
  };

  await saveMeal(meal);
  redirect("/meals");
}

```

6. We can use `useFormStatus` hook in `Next.js`
   - Just like in react we have to create a separate component to use it
   - Then use that component in the form
   - The component where we are using it needs to be a client side component

```
'use client';

import { useFormStatus } from "react-dom";

export default function MealsFormSubmit() {
    const {pending} = useFormStatus();


    return <button type="submit" disabled={pending}>
        {pending ? 'Submitting...' : 'Share Meal'}
    </button>
}

```

7. Server side validations
   - on Submitting of the form we can get the data using the `formData`
   - Then we can add validation on that
   - If we throw an error from a `page.js`, it will go to the closest `error.js` page
   - Problem with this approach is that, even if we missed 1 validation, the whole form is lost and we moved to error page. Ideal behavior should be to show the error on the same form

```example.js
  if(!meal.title || meal.title.trim() === '') {
    throw new Error('Invalid input');
  }
```

8. `useFormState` hook
   - Instead of sending an error in case of validation failure, we can send an object (serializable)
   - Then in the form component we can use `useFormState` hook
   - It takes 2 arguments, 
     - first one is the method that we currently mapped at the action property of form
     - Second is the default value of the that method that we mapped as a first argument
   - It also returns an array of exactly 2 values
     - The latest state (object that we return from the method)
     - Updated method state, now we have to map it to the action property of the form
   - We also need to change the structure of the action function, now it will take 2 arguments
     - First one is the `prevState`
     - Second one is the `formData` object

```setting-message.js
export async function shareMeal(prevValue, formData) {
  const meal = {
    title: formData.get("title"),
    summary: formData.get("summary"),
    instuctions: formData.get("instructions"),
    image: formData.get("image"),
    creator: formData.get("name"),
    creator_email: formData.get("email"),
  };

  if(!meal.title || meal.title.trim() === '') {
    return {
        message: 'Invalid input'
    }
  }

  await saveMeal(meal);
  redirect("/meals");
}

```

```page.js
export default function ShareMealPage() {
  const [state, formAction] = useFormState(shareMeal, {message: null});

  return (
    <>
      <header className={classes.header}>
        <h1>

        </h1>
        <p>Or any other meal you feel needs sharing!</p>

            <p>
                <label htmlFor="title">Title</label>
                <input type="text" id="title" name="title" required />
            </p>
			...
            {state.message && <p>{state.message}</p>}
        </form>

    </>
  );
}
```
